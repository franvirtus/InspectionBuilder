# InspectionBuilder Design Rules

InspectionBuilder should look like a premium professional SaaS for home inspection reports.

Design language:
- dark background
- elevated panels
- soft borders
- blue primary action
- slate secondary action
- red danger action
- severity colors preserved
- compact but readable cards
- strong section headings
- clear information density

Avoid:
- default white Streamlit look
- excessive gradients
- childish visuals
- too many icons
- random colors
- inconsistent button styles
- large empty spaces
- excessive click depth

Preferred UI components:
- dashboard summary cards
- sidebar report context
- defect cards
- severity badges
- form sections
- clear primary CTA
- sticky-feeling report actions where possible within Streamlit limits
